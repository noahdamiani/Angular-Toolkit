1R Toolkit / Angular Playground

  1. Branching
    • Create a new branch for your module, do not work on master.
    • When the module is complete, put up a code review to get feedback, then we can push it.
  2. Create Module
    • Create a folder for your module in the /modules directory. The goal is to keep your module completely self contained.
    • Add a view, stylesheet, and controller to get started (myModuleName.html, myModuleName.css, myModuleName.js)
    • Link your module in the header and footer of index.html
    • In myModuleName.js, add the following:
      app.controller('MyModuleNameController', function ($scope) { });
    • Open up router.js and create a new .state() following the other examples. Put the correct path for your module view in the templateUrl key. In the controller, put the name of your controller ie. MyModuleNameController.
  3. Adding a completed module to Side Nav
    • Head over to index.html and find the .side-nav class. Add a list item and anchor tag with the ui-sref attribute. Your target will be the name of your module as defined in the router state. Example: .state('notepad', {}); --> ui-sref="notepad".
  4. Working with FireBase.com
    • Shoot an email to noah@onerockwell.com to get the firebase credentials.
    • You will need to login, and go to the auth tab in the admin. From there add a local domain (wherever you are hosting your toolkit environment). OR the easy way: use dev.1r-toolkit.com in your v-host and you will already be authorized.
    • From there, you will have access to the FireBase DB for toolkit. There is currently an angular factory setup for instant use. Inject FireBase into your app.controller and add something like $scope.helloWorld = FireBase.getArray('/yourModuleName'). This will access that DB and JSON set. You can add a JSON doc manually in FireBase and you should be able to see the object when doing console.log($scope.helloWorld).
  5. VENDOR DEPENDENCIES
    • Be sure to use Bower to add any dependencies you need. That keeps them neat in the bower_components folder. More details and documentation on Bower here.
